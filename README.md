# おかログ Phase 1 セットアップ手順

このフォルダには、検索・一覧・地図連動画面（Phase 1 MVP）一式が入っています。
上から順番に進めれば、動くサイトが公開できます。分からない箇所が出てきたら、
そこで止めて質問してください。一緒に確認します。

---

## STEP 1. Firebaseプロジェクトを作る

1. https://console.firebase.google.com/ を開き、「プロジェクトを追加」
2. プロジェクト名は「okalog」など好きな名前でOK
3. 作成できたら、左メニュー「構築」→「Firestore Database」→「データベースの作成」
   - ロケーションは `asia-northeast1`（東京）を選択
   - モードは「本番環境モード」を選択
4. 左メニュー「構築」→「Authentication」→「Sign-in method」タブ
   - 「Google」を選んで有効にする（これが後で使うログイン機能です）
5. 左メニューの歯車アイコン→「プロジェクトの設定」→下にスクロールして
   「マイアプリ」→ `</>`（ウェブ）のアイコンをクリックしてアプリを登録
   - 登録すると `firebaseConfig = { apiKey: ... }` のようなコードが表示されます
   - この中身を、お渡しした `firebase-init.js` の `firebaseConfig` の部分に
     そのままコピー＆ペーストしてください

**ここで一度、私に「STEP1終わった」と教えてください。設定値の貼り付けが
うまくいっているか確認します。**

---

## STEP 2. Google Maps APIキーを取得する

1. https://console.cloud.google.com/ を開く（Firebaseと同じGoogleアカウントでOK）
2. 上部のプロジェクト選択で、STEP1で作ったFirebaseプロジェクトを選ぶ
   （FirebaseプロジェクトはGoogle Cloudのプロジェクトと同じものです）
3. 「APIとサービス」→「ライブラリ」で以下の2つを検索して有効化する
   - **Maps JavaScript API**
   - **Geocoding API**
4. 初めて有効化する際に「お支払い情報の登録（クレジットカード）」を求められます。
   - ここは少し不安に感じるかもしれませんが、Googleは毎月一定額の無料枠を
     提供しており、個人運営の規模（月間数千アクセス程度）では通常無料枠内に
     収まります。心配な場合は、Google Cloud Console の「お支払い」画面から
     **予算アラート**（例：1,000円を超えたら通知）を設定しておくと安心です。
5. 「APIとサービス」→「認証情報」→「認証情報を作成」→「APIキー」
   - 作成されたキーを `firebase-init.js` の `GOOGLE_MAPS_API_KEY` に貼り付け
   - セキュリティのため、キーの「アプリケーションの制限」で
     「ウェブサイト」を選び、後述のVercelの公開URLを登録しておいてください
     （STEP4でURLが決まってから設定でOKです）

---

## STEP 3. Firestoreのセキュリティルールを設定する

1. Firebaseコンソール →「Firestore Database」→「ルール」タブ
2. お渡しした `firestore.rules` の中身を全部コピーして貼り付け
3. 「公開」ボタンを押す

（これは「誰が読み書きできるか」を決めるファイルです。今の設定は
「閲覧は誰でもOK、書き込みはログインしていればOK」という暫定ルールです。
Phase 2で編集者・オーナーだけに絞り込みます。）

---

## STEP 4. 自分のお店データを取り込む

1. お手元のExcelの店舗リストを開き、`store-template.csv` と同じ並び順の
   列（name, genre, prefecture, city, area, address, taste, scenes, features, url）
   になるように整えてください
   - genre は20種類のジャンル（お渡しした一覧）から選んで入力
   - area は「梅田・北新地・中之島」「心斎橋・難波・道頓堀」等、決まったエリア名で入力
   - scenes は複数ある場合「デート;接待」のようにセミコロン区切り
2. 「名前を付けて保存」→ ファイル形式で「CSV UTF-8」を選んで保存
3. パソコンで `public/import.html` をダブルクリックして開く
   （ブラウザで file:// として開いてもOKですが、うまく動かない場合は
   STEP5のVercel公開後に、公開されたURLの `/import.html` を開いてください）
4. 「Googleでログイン」→ 自分のアカウントでログイン
5. 作成したCSVファイルを選択 →「Firestoreに取り込む」
6. 画面下のログに「✔ 店名を登録しました」が並べば成功です

---

## STEP 5. GitHub + Vercelで公開する

すでにアカウントをお持ちとのことなので、手短にポイントだけ：

1. このフォルダの中身（`public`フォルダの中身一式）をGitHubの新しいリポジトリにpush
2. Vercelでそのリポジトリを「Import」
   - Root Directory（公開するフォルダ）は `public` を指定
   - ビルドコマンドは不要（静的ファイルなのでそのまま公開されます）
3. デプロイが終わると `https://xxxx.vercel.app` のようなURLが発行されます
4. STEP2で後回しにしたMaps APIキーの「ウェブサイト制限」に、このURLを追加してください

これでスマホ・PCどちらからも検索・地図連動が動くはずです。

## 動作確認のポイント

- 検索画面（`index.html`）を開いて、都道府県→市区町村→エリアの絞り込みが
  連動して変わるか
- ジャンル・シーンのチップをクリックすると一覧が絞り込まれるか
- 店舗カードをタップすると地図のピンが移動するか（スマホでは地図まで自動スクロールするか）
- 何も表示されない場合は、まず `import.html` でデータが登録できているか、
  ブラウザの「開発者ツール→コンソール」にエラーが出ていないかを確認してください

## STEP 6. 自分を「オーナー」に設定する（最初の1回だけ必要）

権限（オーナー／編集者／ログインユーザー）の仕組み上、**最初のオーナーだけは
Firebaseコンソールから手動で設定する必要があります。** これは安全のための
仕様で、一度やれば以降は不要です。

1. まず `index.html`（公開サイトのトップページ）を開き、右上の
   「Googleでログイン」で、ご自身のGoogleアカウントでログインする
   （これで自動的に`role: "member"`のアカウントが作られます）
2. Firebaseコンソール →「Firestore Database」→「データ」タブを開く
3. `users` というコレクションを開くと、今ログインしたご自身のアカウントの
   ドキュメントが1件あるはずです（メールアドレスで見分けられます）
4. そのドキュメントを開き、`role` というフィールドの値を
   `member` から **`owner`** に書き換えて保存
5. サイトに戻って再読み込みすると、「オーナー」と表示され、
   「＋ お店を追加」ボタンや、各店舗の「編集」「削除」ボタンが
   使えるようになります

### 編集者を追加したい場合

信頼できる人にGoogleアカウントでログインしてもらい（一度サイトを開いて
ログインするだけでOKです）、同じように`users`コレクションから
その人のドキュメントを探し、`role`を`editor`に書き換えてください。

- **編集者ができること**：お店の追加・編集、削除依頼
- **編集者ができないこと**：削除の実行（オーナーの承認が必要）

### 削除依頼の確認方法

編集者が「削除依頼」をすると、該当のお店カードに赤い「削除依頼あり」の
表示が出ます（これはオーナーにしか見えません。依頼した本人には「削除依頼中」、
それ以外の人には何も表示されません）。トップページを見て、その表示がある
お店をオーナーが確認のうえ「削除」（実行）または「依頼を却下」（取り消し）
ボタンで対応してください。

## STEP 7. 削除依頼をメールで通知する（EmailJSの設定）

編集者が削除依頼を出すと、あなたのメールアドレスに自動でメールが届くように
します。Firebaseの標準機能だけでは実現できないため、**EmailJS**という
無料の外部サービス（クレジットカード登録不要、月200通まで無料）を使います。

1. https://www.emailjs.com/ にアクセスし、「Sign Up」で無料アカウントを作成
2. 左メニュー「Email Services」→「Add New Service」→ **Gmail** を選択し、
   ご自身のGmailアカウントで連携（許可を求められたら許可）
   - 作成されると **Service ID**（`service_xxxxxxx`のような文字列）が発行されます
3. 左メニュー「Email Templates」→「Create New Template」
   - **To Email**：ここに、削除依頼を受け取りたいご自身のメールアドレスを入力
     （これはEmailJSの管理画面上だけに保存され、コードには書きません）
   - **Subject**：`【おかログ】削除依頼がありました：{{store_name}}`
   - **Content**（本文）：
     ```
     {{requester_name}} さんが「{{store_name}}」の削除を依頼しました。
     日時：{{request_date}}

     サイトで確認してください：https://okalog2.vercel.app
     ```
   - 保存すると **Template ID**（`template_xxxxxxx`のような文字列）が発行されます
4. 左上のアカウントアイコン →「General」→ **Public Key** をコピー
5. 念のためのセキュリティ設定：「Account」→「Security」で
   「Allowed origins（許可するドメイン）」に
   ```
   https://okalog2.vercel.app
   ```
   を追加しておくと、他のサイトからこの設定を勝手に使われるのを防げます
6. 取得した3つの値（Public Key、Service ID、Template ID）を、
   `firebase-init.js`の該当箇所に貼り付けてGitHubにアップロード

これで、編集者が「削除依頼」ボタンを押すたびに、あなたのメールに
自動で通知が届くようになります。

## STEP 9. 行きたい店リスト機能について

`wishlist.html` を新しく追加しました。ログイン中の人には、検索画面（`index.html`）
の右上に「行きたい店リスト」というリンクが表示されます。

- ログインしていれば誰でも、行きたいお店を登録できます
- 登録時に「匿名で登録する」にチェックを入れると、他の人には誰が登録したか
  表示されません（本人だけは自分の登録と分かります）
- 登録された行きたい店を見た人が、実際に行ってみたら「感想」を書き込めます
  （感想を書いた人の名前は表示されます）
- 削除できるのは、登録した本人か、編集者・オーナーだけです

このページも`firestore.rules`の更新が必要です（`wishlist`コレクション用の
ルールを追加済みです）。前回同様、Firebaseコンソールの「ルール」タブに
新しい内容を貼り付けて公開してください。

## STEP 10. 行きたい店リストへの感想通知（もう1つEmailJSテンプレートを作る）

STEP 7で作った削除依頼通知は「あなた（オーナー）宛て」固定でしたが、今回は
「登録した人ごとに違う宛先」へ送る必要があるため、**新しいテンプレートを
もう1つ**作成します。EmailJSのアカウントはSTEP 7で作ったものをそのまま使います。

1. EmailJS（https://www.emailjs.com/）にログイン
2. 左メニュー「Email Templates」→「Create New Template」
   - **To Email**：`{{to_email}}` と入力（固定のメールアドレスではなく、
     プログラム側から毎回渡す形にします）
   - **Subject**：
     ```
     【おかログ】「{{store_name}}」に感想が届きました
     ```
   - **Content**（本文）：
     ```
     {{commenter_name}} さんが、あなたが登録した「{{store_name}}」に
     行ってきて、感想を書いてくれました。

     感想：{{impression_text}}

     サイトで確認してください：https://okalog2.vercel.app/wishlist.html
     ```
   - From Name：`おかログ通知`
   - From Email：「Use Default Email Address」にチェック
3. 保存すると新しい **Template ID**（`template_xxxxxxx`）が発行されます
4. その値を`firebase-init.js`の`EMAILJS_WISHLIST_TEMPLATE_ID`に貼り付けて、
   GitHubにアップロード

これで、誰かが行きたい店リストに感想を投稿すると、登録した本人（匿名で
登録していても、本人にはちゃんと）にメールが届くようになります。
なお、登録した本人が自分で感想を書いた場合は通知は届きません（自分への
通知は不要なため、自動でスキップされます）。
