# おかログ Phase 1 セットアップ手順

このフォルダには、検索・一覧・地図連動画面（Phase 1 MVP）一式が入っています。
上から順番に進めれば、動くサイトが公開できます。分からない箇所が出てきたら、
そこで止めて質問してください。一緒に確認します。

---

## STEP 1. Firebaseプロジェクトを作る

1. https://console.firebase.google.com/ を開き、「プロジェクトを追加」
2. プロジェクト名は「okalog」など好きな名前でOK
3. 作成できたら、左メニュー「構築」→「Firestore Database」→「データベースの作成」
   - ロケーションは `asia-northeast1`（東京）を選択
   - モードは「本番環境モード」を選択
4. 左メニュー「構築」→「Authentication」→「Sign-in method」タブ
   - 「Google」を選んで有効にする（これが後で使うログイン機能です）
5. 左メニューの歯車アイコン→「プロジェクトの設定」→下にスクロールして
   「マイアプリ」→ `</>`（ウェブ）のアイコンをクリックしてアプリを登録
   - 登録すると `firebaseConfig = { apiKey: ... }` のようなコードが表示されます
   - この中身を、お渡しした `firebase-init.js` の `firebaseConfig` の部分に
     そのままコピー＆ペーストしてください

**ここで一度、私に「STEP1終わった」と教えてください。設定値の貼り付けが
うまくいっているか確認します。**

---

## STEP 2. Google Maps APIキーを取得する

1. https://console.cloud.google.com/ を開く（Firebaseと同じGoogleアカウントでOK）
2. 上部のプロジェクト選択で、STEP1で作ったFirebaseプロジェクトを選ぶ
   （FirebaseプロジェクトはGoogle Cloudのプロジェクトと同じものです）
3. 「APIとサービス」→「ライブラリ」で以下の2つを検索して有効化する
   - **Maps JavaScript API**
   - **Geocoding API**
4. 初めて有効化する際に「お支払い情報の登録（クレジットカード）」を求められます。
   - ここは少し不安に感じるかもしれませんが、Googleは毎月一定額の無料枠を
     提供しており、個人運営の規模（月間数千アクセス程度）では通常無料枠内に
     収まります。心配な場合は、Google Cloud Console の「お支払い」画面から
     **予算アラート**（例：1,000円を超えたら通知）を設定しておくと安心です。
5. 「APIとサービス」→「認証情報」→「認証情報を作成」→「APIキー」
   - 作成されたキーを `firebase-init.js` の `GOOGLE_MAPS_API_KEY` に貼り付け
   - セキュリティのため、キーの「アプリケーションの制限」で
     「ウェブサイト」を選び、後述のVercelの公開URLを登録しておいてください
     （STEP4でURLが決まってから設定でOKです）

---

## STEP 3. Firestoreのセキュリティルールを設定する

1. Firebaseコンソール →「Firestore Database」→「ルール」タブ
2. お渡しした `firestore.rules` の中身を全部コピーして貼り付け
3. 「公開」ボタンを押す

（これは「誰が読み書きできるか」を決めるファイルです。今の設定は
「閲覧は誰でもOK、書き込みはログインしていればOK」という暫定ルールです。
Phase 2で編集者・オーナーだけに絞り込みます。）

---

## STEP 4. 自分のお店データを取り込む

1. お手元のExcelの店舗リストを開き、`store-template.csv` と同じ並び順の
   列（name, genre, prefecture, city, area, address, taste, scenes, features, url）
   になるように整えてください
   - genre は20種類のジャンル（お渡しした一覧）から選んで入力
   - area は「梅田・北新地・中之島」「心斎橋・難波・道頓堀」等、決まったエリア名で入力
   - scenes は複数ある場合「デート;接待」のようにセミコロン区切り
2. 「名前を付けて保存」→ ファイル形式で「CSV UTF-8」を選んで保存
3. パソコンで `public/import.html` をダブルクリックして開く
   （ブラウザで file:// として開いてもOKですが、うまく動かない場合は
   STEP5のVercel公開後に、公開されたURLの `/import.html` を開いてください）
4. 「Googleでログイン」→ 自分のアカウントでログイン
5. 作成したCSVファイルを選択 →「Firestoreに取り込む」
6. 画面下のログに「✔ 店名を登録しました」が並べば成功です

---

## STEP 5. GitHub + Vercelで公開する

すでにアカウントをお持ちとのことなので、手短にポイントだけ：

1. このフォルダの中身（`public`フォルダの中身一式）をGitHubの新しいリポジトリにpush
2. Vercelでそのリポジトリを「Import」
   - Root Directory（公開するフォルダ）は `public` を指定
   - ビルドコマンドは不要（静的ファイルなのでそのまま公開されます）
3. デプロイが終わると `https://xxxx.vercel.app` のようなURLが発行されます
4. STEP2で後回しにしたMaps APIキーの「ウェブサイト制限」に、このURLを追加してください

これでスマホ・PCどちらからも検索・地図連動が動くはずです。

## 動作確認のポイント

- 検索画面（`index.html`）を開いて、都道府県→市区町村→エリアの絞り込みが
  連動して変わるか
- ジャンル・シーンのチップをクリックすると一覧が絞り込まれるか
- 店舗カードをタップすると地図のピンが移動するか（スマホでは地図まで自動スクロールするか）
- 何も表示されない場合は、まず `import.html` でデータが登録できているか、
  ブラウザの「開発者ツール→コンソール」にエラーが出ていないかを確認してください
